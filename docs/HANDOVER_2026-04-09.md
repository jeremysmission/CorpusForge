## What changed in CorpusForge

### Bug fixes (reviewer, this session)

1. **`src/embed/embedder.py`** — CUDA_VISIBLE_DEVICES remapping fix.
   When the env var is set to a non-zero physical GPU index, the embedder previously tried `cuda:N` where N was the env value. PyTorch had already remapped the visible GPU to `cuda:0`, so `cuda:N` raised "invalid device ordinal". Fix: read the env var separately for logging (physical GPU number), but always use `cuda:0` for the device string. No regression test added.

2. **`src/pipeline.py:_embed_chunks()`** — sub-batch embedding for large corpora.
   When total chunks > 100,000, the embedder now processes in 100K sub-batches and writes vectors to a memory-mapped temp file before copying into a final float16 array, instead of accumulating all vectors as a list of float32 arrays in RAM. Goal: prevent OOM during 2M+ chunk runs. No dedicated regression test (the test corpus is small).

3. **`src/parse/parsers/archive_parser.py`** — ARCHIVE-DEFER LEAK FIX (the big one).
   - Added `deferred_exts` parameter to `__init__` (set of extensions, normalized to dotted lowercase form).
   - Added `_is_name_deferred(name)` helper that does **segment-based** matching: split the lowercased basename on `.`, check if any dot-segment equals a deferred token. This catches both `foo.SAO.XML` (token mid-name) and `foo.sao` (token at end), but does NOT over-match `report_sao.log` or `bigsao.txt`.
   - In `parse()` entry: if the archive's OWN basename has a deferred dot-segment, return an empty doc immediately and skip extraction entirely. This is the critical fix for production data where archives are named `AS00Q_2015309093000.SAO.zip` and contain `AS00Q_2015309093000.SAO.XML` members.
   - In `_extract_zip()` and `_extract_tar()`: each member's original (in-archive) name is checked against `_is_name_deferred` BEFORE the temp file is written. The check uses the original member name, not the extracted temp filename, so the producer naming convention is preserved.
   - The hardcoded `_SKIP_EXTENSIONS` set (which protects against nested archive recursion) is unchanged — it remains in place alongside the new defer policy.

4. **`src/parse/dispatcher.py`** — wired the defer set through.
   - `_build_parser_map(skip_list_path, extra_deferred_exts=None)` now loads `deferred_formats` from `skip_list.yaml` (canonical source #1) AND merges in `extra_deferred_exts` (canonical source #2, comes from `config.parse.defer_extensions`). The combined set is passed once to `ArchiveParser(deferred_exts=...)`. No new local constant.
   - `ParseDispatcher.__init__` accepts `extra_deferred_exts` and passes it to `_build_parser_map` on first lazy build.
   - Added `reset_parser_map()` so the cached module-global `_PARSER_MAP` can be invalidated when defer policy changes between runs (used by the pipeline init and by the regression tests).

5. **`src/pipeline.py:Pipeline.__init__`** — wired config defer policy through to the dispatcher.
   - Calls `reset_parser_map()` so any cached parser map from a prior run does not stick around with stale defer state.
   - Constructs `ParseDispatcher(..., extra_deferred_exts=set(config.parse.defer_extensions))`.

## What changed in HybridRAG V2

### V2 import-side filter (morning safety net)

`scripts/import_embedengine.py` now accepts `--exclude-source-glob GLOB` (repeatable). Implementation lives in two new functions in the same file:

- `_matches_any_glob(source_path, patterns)` — case-insensitive fnmatch check against the basename, the full path, and each path component.
- `apply_exclude_source_globs(chunks, vectors, exclude_globs)` — drops matching chunks and the corresponding vector rows, returns `(kept_chunks, kept_vectors, excluded_count)`. Handles `np.memmap` correctly by materializing a regular array for the indexed slice.

The `main()` block:
- Adds the `--exclude-source-glob` CLI arg with help text that calls it out as a temporary Sprint 6.6 fallback.
- Prints an explicit banner at startup when the filter is active (or "No --exclude-source-glob filter active." when it isn't).
- Prints a result line: `Filter result: N excluded, M kept (pre -> post)`.
- Mutates the manifest dict (in-memory) to add an `import_filters` block with the active globs, pre/post counts, excluded count, reason, and retire-when condition. This propagates into any downstream report writer that copies manifest fields.

**Verified against the Run 5 export:**
